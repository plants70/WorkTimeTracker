
# CONTRIBUTING

## Как работать с задачами
- Для каждой задачи создавайте отдельный Issue.
- В Issue указывайте контекст, требование, критерии готовности, места правок.
- Ссылки на код и листы Google Sheets приветствуются.

## Ветки и PR
- Имена веток: `feat/...`, `fix/...`, `chore/...`, `refactor/...`.
- Коммиты: `type(scope): кратко` (например, `fix(sessions): prevent duplicate logout`).
- PR — небольшой размер, чек-лист проверки и скриншоты/логи при необходимости.

## Качество
- Формат кода: `black`.
- Линтинг: `ruff`.
- Тесты: `pytest` (моки для внешних сервисов).
